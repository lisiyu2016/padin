Basics and understanding

Here we are describing some basics about the dashboard and explain some definitions. You can connect your projects to pixelplux.com dashboard for realtime monitoring, artificial intelligence, and reporting. We provide scalable and relaible platform for Iot projects which can be used for vast variety of purposes.

Integrating projects and softwares

We are trying to provide couple of methods that gives developers this ability to conenct their projects to dashboard. These methods are consisting:

  • RESTful api for biodirectional information, such as temperature projects
  • UDP api for biodirectional information, such as temperature projects
  • MQTT api, for multidirectional communications, such as sending information and recieving orders
Note: UDP and MQTT are still in experimental area, so they are not available in production and their api and domain access might change in near future.

Getting a token

You can find your Access Token inside your settings section. Make sure you use the Access Token only to update us for your project information, such as the changes in temperature or humidity, projects usages and etc. There might be other tokens provided to you and each have their own usage. Eversince you have your token, you can make post and get requests to the api server included x-token in headers.

CloudProject vs DataSource

We have two definition in dashboard for data and projects which are really simple to understand and straightforward.

CloudProject is a project that you can create in your management dashboard, in your Projects > Create a project. This project represents a phyical project or a part of that, and main purpose is to be used for human interactions, drawing charts and making reports. Each project, has a datasource. This datasource, is a raw data, such as a number for temperature that you send to api from your project. All projects must be connected to a datasource inorder to collect information.

When you send your data with a data source id, we collect it, but if there is no project for this API we will show it in projects pages, as unconnected project. We are not subject to hold these information for long, if there is no project assigned. So it's necessary to create project based on project as quick as possible.

You can connect a datasource to many projects only for biodirectional projects. We don't know why you might use this, but it's available in case of any usages.
